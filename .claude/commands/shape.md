To have a better understanding of the application current state we will maintain uml diagrams in mermaidjs format

- Use git to get the current changes
- Only consider changes inside app/
- Identify what classes are involved in the git changes
- Add or update them in the mermaidjs file uml/classes.mmd
- Add or update a sequence diagram for each of the potentially affected routes in a mmd diagram inside uml/routes/
